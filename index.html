Comandos Úteis do Git HUB

O GitHub é uma plataforma de compartilhamento de código-fonte que facilita muito o trabalho de um desenvolvedor, principalmente quando se trabalha em grupo

Abaixo, listaremos alguns comandos úteis para utilizar quando for trabalhar com Git Hub .


Nome do Comando Descrição Exemplo
init Cria um novo repositório Git vazio na pasta atual. git init
clone Cria uma cópia local de um repositório remoto. git clone endereço_do_repositório_a_ser_clonado
branch Cria, lista ou exclui ramificações (branches) no repositório. git branch teste (cria uma branch chamada "teste")
checkout Alterna entre branches, restaura arquivos modificados ou criar novas branches de desenvolvimento. git checkout teste (alterna para a branch "teste")
status Exibe o estado atual do repositório, mostrando arquivos modificados, adicionados, excluídos ou não rastreados. git status
diff Mostra as diferenças entre os arquivos no diretório de trabalho e a área de preparação (staging area). git diff nome_do_arquivo
add Adiciona as alterações nos arquivos ao índice do Git, preparando-os para serem commitados posteriormente. git add . (Adiciona todas as alterações que você fez no projeto.)
commit Salva as alterações realizadas em um repositório como uma nova versão. git commit -m "Descrição do Commit"
push Envia as alterações locais em um repositório para um repositório remoto, como o GitHub. git push origin nome_da_branch
pull Busca as alterações remotas e as mescla com a versão local do repositório. git pull origin master
revert Desfaz um ou mais commits anteriores, criando um novo commit que reverte as alterações introduzidas por esses commits. git revert nome_do_commit
merge Combina as alterações de diferentes branches em uma única branch. git merge feature (Nota: Importante estar na branch main
stash Salva temporariamente alterações não comprometidas. git branch add
rm Remove arquivos do controle de versão do Git e do diretório de trabalho. git rm nome_do_arquivo
config Permite configurar as opções globais ou específicas do repositório do Git. git config --global user.name "Seu Nome"
reset Reverte as alterações em um repositório, desfazendo commits e reposicionando o HEAD e o branch atual. git reset HEAD~1
remote Gerencia e visualiza os repositórios remotos associados a um repositório local do Git. git remote add origin https://github.com/seu-usuario/seu-repositorio.git
fetch Baixa as alterações mais recentes do repositório remoto, mas não faz merge com o branch local. git branch add
show Faz uma descrição de algum outro comando do Git git fetch origin
help Faz uma descrição de algum outro comando do Git git help add
rebase Reescreve o histórico do branch, movendo as alterações para uma nova base de ramificação. git rebase main
blame Exibe a autoria e a última alteração de cada linha de um arquivo específico. git branch add
tag Cria, lista ou exclui tags (marcadores) em um repositório do Git. git tag v1.0.0

Como fazer a configuração e o acesso ao repositório GitHub.

Para configurar e acessar o seu repositório do GitHub em seu computador é necessário seguir alguns passos muito simples:

  1. Com o Git já instalado em sua máquina, abra o terminal e execute os seguintes comandos de configuração:

    git config --global user.name "Seu Nome"

    git config --global user.email seu-email@exemplo.com

  2. No repositório recém-criado no GitHub, clique no botão "Code" (ou "Código") verde e copie o URL do repositório. Depois disso, execute o seguinte código:

    git clone URL_DO_REPOSITÓRIO

Agora você pode criar, modificar e excluir arquivos dentro do diretório do repositório no seu computador.
